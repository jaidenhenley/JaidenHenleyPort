Challenges & Solutions

OCR Messiness

Challenge: Scanned notes can produce inconsistent OCR output.
Fix: Added a cleanup pass before generation.
Result: Cleaner flashcards and less manual correction.

AI Availability and Failures

Challenge: AI generation is not always available or can fail.
Fix: Added deterministic fallback generation plus clear error UI.
Result: Reliable card generation behavior across devices.

Noisy Generated Cards

Challenge: Not every generated flashcard is worth studying.
Fix: Added approval toggles so cards can be enabled/disabled.
Result: Users only practice high-value cards.

Core Features

Primary product capabilities included in QuickStudy.

Study Flow

- Scan notes or import PDFs
- Clean-up options (handwriting mode/spellcheck when enabled)
- Generate flashcards automatically
- Approve-only studying (toggle cards on/off)
- Flashcard practice loop
- Quiz mode with distractors
- Save and reload study sets

Import to Cards

Input: scanned note or PDF
1. Extract text
2. Run cleanup pass
3. Generate card candidates
4. Save generated set for review

Approval Controls

1. Present generated cards
2. Toggle card include/exclude
3. Build active study deck from approved cards
4. Route approved deck to practice or quiz

Persistence

SavedSets.json stores:
- Deck metadata
- Approved card set
- Quiz-ready data
- Reload state for follow-up sessions

Tech Stack

  • SwiftUI
  • Apple Intelligence (FoundationModels)
  • PDF import
  • Local persistence (SavedSets.json)

Outcome

QuickStudy made it faster to move from raw notes to active recall by combining import, generation, approval, and practice in one focused flow.
