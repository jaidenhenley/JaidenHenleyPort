# Jaiden Henley Portfolio

A personal portfolio built to showcase my iOS projects, case studies, and the work I’ve done through the Apple Developer Academy and outside of it. I wanted one place that felt more complete than a resume and gave people a better sense of how I build, think through products, and present my work.

[View Live Portfolio](https://jaidenhenley.github.io/JaidenHenleyPort/)

## Stack

HTML, CSS, JavaScript

## Features

- Responsive portfolio site built to present my work clearly across devices
- Featured project sections for apps like QuickStudy, TakeFlight, and CommonSight
- Individual case study pages that go deeper into my role, process, and decisions
- Resume, contact links, and project access all in one place
- Custom styling and layout built to feel more personal and reflect my brand
- Space to keep updating projects as I build more work

## Architecture

This is a front-end portfolio project built to organize and present my work in a way that feels polished and easy to navigate.

**Project showcase.** The site is structured around featured projects and supporting case studies so each project can be understood beyond just a title and screenshot. I wanted the portfolio to show not just what I built, but how I approached the work and what kind of thinking went into it.

**Case study structure.** Each project page follows a more detailed layout that gives room for screenshots, descriptions, challenges, solutions, and outcomes. That made the portfolio more useful as both a personal site and a place to present work professionally.

**Design and layout.** The site uses a custom HTML, CSS, and JavaScript structure to keep the experience simple, clean, and focused on the content. A big part of the goal was making sure the design supported the work instead of distracting from it.

**Scalability.** The portfolio was set up in a way that makes it easy to add new projects, update existing case studies, and keep refining the overall presentation as I grow. It is something I plan to keep building on rather than treating as a one-time project.
